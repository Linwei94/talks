Background
Not a Chatbot

The difference is who does the work

ChatGPT / Copilot

You copy an error → it suggests a fix
You paste it back → you run it
You copy the new error…
You are the middleman.

Claude Code

You describe the goal
It reads files, writes code, runs it
It fixes errors and loops back
You stay at the level of intent.

Runs in your terminal. Reads and edits your actual files. Loops until done — or asks when stuck.

What It Can Do
Research Tools, Built in Hours

Anything that would normally take days of scripting

Data analysis pipeline
Load raw data → fit → plot → export figures. You describe the model. It writes the code.
Simulation wrapper
Submit jobs, monitor progress, parse output, collect results — automated end to end.
Legacy code migration
Old Fortran or MATLAB scripts → clean Python, with comments and unit tests added.
Lab dashboard
A simple web page to track experiments, samples, or equipment bookings. No web dev skills needed.
Paper draft
Given your results and figures, draft a Methods or Results section in the journal's style.
Debug anything
Stack trace or wrong output — it reads the code, finds the issue, fixes it, and reruns.

I built a complete production web app for a small business in a few days — with little web development experience. The same logic applies to any research tool you need.

Background
How It Actually Works
The LLM

Predicts the most likely next word — over and over. Trained on most of the text humanity has written. Surprisingly good at reasoning as a side effect.

The Agent

LLM + tools + a loop. The model decides what to do, calls a tool (read a file, run code…), sees the result, and keeps going until done.

06 / 11
How It Works
The Context Window

Everything it can see at once

What fits in the window
📄 Your instructions & question
📂 All the files in your project
💬 The entire conversation so far
⚙️ Terminal output, error logs
~200,000 tokens ≈ a small book

Large enough to hold an entire codebase or a full paper draft — all at once.

No memory between sessions

Each new session starts blank. A CLAUDE.md file tells it who you are and what matters — every time.

The latest models support up to 1 million tokens — enough for a full PhD thesis, or months of lab notes, in a single session.

Honest Assessment
What It Does and Doesn't Do

Good at implementation. Not a scientist.

Genuinely useful
✓Writing and debugging code you describe in words
✓Fitting, plotting, and summarising datasets
✓Migrating legacy Fortran / MATLAB to Python
✓Drafting Methods and Results sections
✓Formatting references and fixing LaTeX
Still needs you
—Deciding which question is worth asking
—Knowing if a result makes physical sense
—Choosing the right model or approximation
—Verifying cited references (it hallucinates)
—Scientific judgement. Full stop.

Think of it as a highly capable lab assistant who is excellent at implementation — but has no domain expertise. You still need to supervise the work.

🔴 Live Demo
Let's try it now

Can we write a physics paper right now?

The gap in the literature

CMB energy injection efficiency feff has only been computed for direct χχ → e⁺e⁻. But when annihilation proceeds through an off-shell dark photon (Boehm's model), feff depends on the mediator mass — and nobody has mapped this out. The blind spot changes the exclusion limits significantly.

Prompt to Claude Code

"Do a literature review on CMB constraints on sub-GeV dark matter annihilating via a light dark photon mediator. Identify a gap or overlooked scenario in the existing literature. Propose a novel research direction, design the calculation needed to address it, implement it, and write up the results as a short paper in LaTeX. Render the final PDF."

You tell me if the physics is right — that's the only part it can't do alone.
