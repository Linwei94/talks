What Is an AI Agent?
Context Engineering

Agent = Feed the Right Info into LLM's Context Window

How LLM Works

// Everything LLM sees on each call context = [ system_prompt, // rules, tool list, memory files conversation, // conversation history tool_results, // previous step results your_prompt, // what you just typed ] output = LLM(context) // predict next token

LLM has no memory — it can only see what's in the context

Skills & Skill-Creator

Skills — Give Your Agent a Custom Workflow

What is a Skill?

A Markdown file describing how the Agent should complete a category of tasks. Once triggered, the Agent follows the skill's instructions to execute the full workflow.

What is Skill-Creator?

A skill specifically for writing skills. Describe your need, and Agent automatically generates a structured workflow spec and saves it.

Use Cases

Literature review, paper writing, experiment management, code review, data preprocessing — any repetitive workflow can be packaged as a skill

Skills = reusable Agent workflows — define once, invoke repeatedly

Examples: /paper-review review workflow · /experiment-log experiment record · /debug-cluster GPU cluster debug · /weekly-report auto weekly report

Tech · Tool Use
Function Calling

How Agent "Acts": The Tool Use Loop

Agent receives task
→
Sent to LLM (with tool list)
→
[tool_use] Read("main.py")
↓
Execute Read("main.py") on computer
→
Return file contents
→
Add to context, call LLM again
↓ Loop until task complete
[tool_use] Write("fix.py", ...)
→
Write to file
→
"done" [END]

Why it's powerful: can execute any shell command — text is what LLM does best, and shell commands are text

Tech · Memory
Memory Architecture

The System Prompt Holds the Agent

  • CLAUDE.md: project rules, coding conventions, notes
  • memory/*.md: user preferences, past decisions, long-term memory
  • skills/*.md: reusable workflows (skill library)
  • Tool list: Read, Write, Bash, WebSearch…
  • At every LLM call, all of this is stuffed into the system prompt
# CLAUDE.md example ## Rules - Reply in English - Check GPU usage before running experiments - Do not commit untested code ## Project Context NeurIPS 2026 submission, ddl: 5/31

A single conversation can load 4000+ tokens of system info —
that's why the agent can continuously "remember" project state

Tech · Sub-agent
Context Engineering

Sub-agent: Solving the Context Bottleneck

Task too big to fit in one context?
The main agent spawns multiple sub-agents that handle subtasks in parallel, returning only summarized results to the main context.

Main Agent
→ spawn
Sub A: Read Paper A
→ spawn
Sub B: Read Paper B
← Returns summary only, not full text
  • Sub-agent has a lean system prompt, focused on a single subtask
  • Context window contains only task-relevant info — more accurate reasoning
  • Multiple sub-agents run in parallel — faster
  • This is the core trick of Context Engineering

When you keep delegating down, who actually does the work? — The bottom-level sub-agent calls tools and actually executes

Tech · Context Compression
From Hongyi Lee's Course

Context Overflow — How to Compress?

Why Does Context Grow So Fast?

Tool outputs (tool observations) account for 84% of context
The model's own words are only ~10%

Two Compression Methods

1

Observation Masking

Replace tool output with a single line:
“There used to be tool output here”

Looks brutal — but experiments show it works about as well as LLM summarization

2

LLM Summarization

History too long → compress it with LLM summarization
OpenClaw has this compaction mechanism built in

Sub-agent = Automatic Compression

Main Agent spawns Sub-agent → Sub-agent accumulates its own Context
↓ sub-agent return
Sub-agent's entire conversation vanishes, only the returned sentence remains

Language models dislike compressing their own memory
So compression is usually enforced by the Agent framework

Discussion · Engineering
Engineering

Senior Engineers 10× Productive — Why Still Hire Juniors?

Managing people vs. managing Agents
Junior hires need onboarding, mentoring, review — and might even pass the blame
Agent follows instructions, doesn't need coddling, always available, no salary

  • One experienced person + agent can do what used to take a team of 5
  • The repetitive work juniors used to do is now fully handled by agents
  • Companies' headcount calculus is being repriced

Do juniors / new grads still have a shot?

  • Yes — but the path is narrower; you need to find your irreplaceable part earlier
  • Systems judgment, user communication, defining what actually needs to be built
  • Those who can command agents are the new era's 10x engineer

This isn't pessimism — it's a new starting line
What used to take 5 years to build end-to-end might now take 1
